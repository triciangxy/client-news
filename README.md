# Client Pulse

Dashboard for tracking news and price moves across our key APAC clients.

## What's in this repo

| File | What it does |
|---|---|
| `index.html` | The dashboard. Reads `data.json` and renders. |
| `data.json` | Cached headlines + quotes. Regenerated by the workflow. |
| `fetch_news.py` | Python script that pulls news + quotes. Runs on GitHub Actions. |
| `.github/workflows/refresh.yml` | Scheduled job that runs `fetch_news.py` and commits the result. |

## How it works

1. Every 30 minutes (during APAC business hours, Mon–Fri), GitHub Actions runs `fetch_news.py` on a free Ubuntu runner.
2. The script fetches stock quotes (Yahoo Finance chart endpoint) and headlines (Google News RSS filtered to each client's domain).
3. It writes results to `data.json` and commits it back to the repo.
4. GitHub Pages serves `index.html`, which reads the latest `data.json` and renders the cards.

No CORS proxies. No paid APIs. No server infrastructure to maintain.

## First-time deployment (one-time setup, ~10 min)

### Step 1 — Create the repo

1. On GitHub, click **New repository**.
2. Name it something unguessable, e.g. `cp-internal-x7k2`. **Public** (Pages on private repos requires a paid plan).
3. Don't add a README or .gitignore — we'll push the files directly.

### Step 2 — Push these files

Drag and drop all four files into the repo on github.com (use **Add file → Upload files**):
- `index.html`
- `data.json` (the starter file with the most recent run included)
- `fetch_news.py`
- `.github/workflows/refresh.yml` (you'll need to type the path manually since GitHub's drag-and-drop doesn't preserve folders — easier: create the file via **Add file → Create new file**, name it `.github/workflows/refresh.yml`, paste the contents)

Commit to the `main` branch.

### Step 3 — Enable GitHub Pages

1. Repo → **Settings** → **Pages** (left sidebar).
2. **Source**: Deploy from a branch.
3. **Branch**: `main`, folder `/ (root)`. Save.
4. After ~30 seconds, the URL appears at the top of the Pages settings: `https://<your-username>.github.io/cp-internal-x7k2/`.

### Step 4 — Grant the workflow permission to push

1. Repo → **Settings** → **Actions** → **General** (left sidebar).
2. Scroll to **Workflow permissions**.
3. Select **Read and write permissions**. Save.

This lets the scheduled job commit updates to `data.json`.

### Step 5 — Trigger the first run manually

1. Repo → **Actions** tab.
2. Click **Refresh Client Pulse Data** on the left.
3. Click **Run workflow** → **Run workflow**.
4. Wait ~30 seconds. It should commit a fresh `data.json`.

Open your Pages URL — you should see the dashboard with live data.

## Editing the client list

Edit the `CLIENTS` list at the top of `fetch_news.py`. Each entry can have:

| Field | Required | Notes |
|---|---|---|
| `name` | yes | Display name shown on the card |
| `short` | yes | Short key used internally |
| `type` | yes | One of: `listed`, `sovereign`, `pension`, `insurance`, `bank` |
| `country` | yes | Display label |
| `ticker` | listed only | Yahoo Finance ticker (e.g. `1299.HK`, `7182.T`, `055550.KS`) |
| `aum` | private only | AUM in USD billions, for display |
| `earnings` | optional | ISO date `YYYY-MM-DD` of next confirmed earnings |
| `blurb` | optional | One-line description shown in the detail drawer |
| `newsroom` | optional | URL to the client's press page |
| `sources` | yes | List of news sources, tried in priority order |

Each source has `type` (badge category), `label` (what shows on the card), `kind` (`google-news` or `rss`), and either `query` (for google-news) or `url` (for rss).

After editing, commit the change. The next scheduled run will pick it up, or trigger it manually from the Actions tab.

## Schedule

The default cron `*/30 0-10 * * 1-5` runs every 30 min from 00:00–10:00 UTC, Monday–Friday (Singapore: 08:00–18:00 SGT). Edit `.github/workflows/refresh.yml` to change it.

GitHub Actions sometimes delays scheduled jobs by a few minutes when their runners are busy — that's normal, not a bug.

## Troubleshooting

**"Could not load data.json" error on the page**
→ The workflow hasn't run yet, or it failed. Check the Actions tab.

**Workflow fails with permission error**
→ You skipped Step 4 above. Go back and grant write permissions.

**Cards show "no source"**
→ The fetcher couldn't reach Google News for that client. Could be transient — try re-running. If persistent, the `site:` filter for that domain may need to be broadened.

**Yahoo quote endpoint stops working**
→ Yahoo occasionally changes endpoints. If quotes go missing, the chart endpoint in `fetch_quote_chart()` is the most-stable replacement; for production-grade reliability, swap to a paid feed (Polygon, IEX Cloud, FMP).
